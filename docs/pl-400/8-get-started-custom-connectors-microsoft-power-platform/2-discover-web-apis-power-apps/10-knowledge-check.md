## Answer the following questions to see what you've learned.

**Question 1:** What is the LEAST relevant while creating a custom connector from Azure API Management?

- Each API in Azure API Management has the **Export** menu to create a custom connector
- Azure API Management offers the feature for the direct creation of a custom connector for Power Apps
- To use the custom connector within Power Apps, you don't need to create a connection for it. *(Correct — a connection is always required)*

---

**Question 2:** There are several ways to create a custom connector for Power Apps. What is NOT a way to create the custom connector?

- Export from Azure API Management
- Import an OpenAPI file
- Import a WSDL file *(Correct — WSDL is not a supported method)*

---

**Question 3:** They discuss the differences between connector and connection. What is NOT correct?

- A connector has information about what the APIs look like.
- A connector has information about how to communicate with the APIs.
- A connector knows the connection details to the APIs, like authentication details. *(Correct — it's the **connection** that holds authentication details, not the connector)*
