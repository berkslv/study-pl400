# Summary

In this module, we discussed the forms and different form types for model-driven apps. We discussed how to configure multiple forms and use different form components.

We also discussed views and grids and showed how to use grids. Finally, we introduced charts and dashboards, to show you how to customize a single screen view of your data.

## Key Takeaways

- **Forms** come in four types: Main, Quick Create, Quick View, and Card — each suited for different use cases
- **Multiple forms** can be configured per table, with form order and security roles controlling which form users see
- **Specialized components** (grids, timelines, iFrames, canvas apps, etc.) enhance the form experience
- **Views** come as Personal, System, or Public — each with different levels of access and editability
- **Editable grids** allow inline editing from views and subgrids without opening individual records
- **Charts** provide interactive visual representations of table data, configurable via the chart editor
- **Dashboards** (multi-stream and single-stream) consolidate data and actions into a single real-time display
