# Configure views overview

Views provide a way to display table data as a list. You can configure views via the **View designer** accessed by selecting **Views** from the **Data experiences** pane in the **Tables** dashboard.

## View Designer Interface

The **View designer** provides an interface similar to the **Form designer**:
- **Left panel**: Table columns
- **Right panel**: Properties panel
- Options to add sorting and filtering
- Ability to name/rename and add a description for your view

### Column Management
- Resize columns in the view designer
- Reorder columns by drag and drop
- Each column has its own properties (select the column then **Edit properties**)
- Sort and filter views from the column header options

## Types of Views

### Personal View
- Individually created and only visible to the person who created it
- Can be shared with others

### System View
- Special views the application depends on
- Exist for system tables or automatically created when you create custom tables
- Only editable by **system admins** or **system customizers**

### Public View
- General purpose views that are customizable
- All app users have access through the view selector
- Can be added to subgrids in a form or as a list in a dashboard
- Customizable by makers
