# Summary

In this module, you learned how variables in Power Apps enable both imperative and declarative programming techniques. You explored how different variable types help you control app behavior, manage state, and improve performance.

You also applied these concepts in a hands-on exercise by building a dynamic collection, storing app-wide values in global variables, and using context variables to control screen-specific UI behavior.

## Key concepts to remember

- Power Apps supports both imperative and declarative logic, allowing you to choose the right approach for each scenario.

- Global variables store values that need to be accessed across multiple screens.

- Context variables store values that are scoped to a single screen, making them ideal for UI state.

- Collections store tabular data in memory and work with most table functions and controls, similar to other tabular data sources.
