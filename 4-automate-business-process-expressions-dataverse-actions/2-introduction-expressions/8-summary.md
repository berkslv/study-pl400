# Summary

> Module: Introduction to expressions in Power Automate | Unit 8 of 8 | 1 minute

When working with Power Automate, you'll use one or more functions to create expressions to get the most out of your data. These functions can be used to retrieve data, change data, evaluate data, and more.

As your flow skills grow, your usage of expressions will grow. Becoming familiar with the various functions available and how to combine them will let you unlock the full power of Power Automate.

## Key takeaways

- **Expressions** are sets of simple functions that return data in the format you need.
- Power Automate and Azure Logic Apps share the same functions — their documentation is interchangeable.
- Use the **fx** button or type `/` in any input field to open the expression editor.
- Function names are **not case-sensitive**.
- Strings in expressions use **single quotes** (`'like this'`).
- Build complex expressions by **nesting functions** — the output of one function becomes the input of another.
- Use **Compose** actions with the **Manually trigger a flow** trigger to safely test expressions.
- Use **notes** on actions to document and preserve your expression formulas.
- **Copilot** can help you create expressions from natural language descriptions.

## Function categories

| Category | Purpose |
|----------|---------|
| String | Modify, search, format text |
| Collection | Work with arrays and strings (length, join, union, etc.) |
| Logical | Conditions and comparisons (if, greater, equals, etc.) |
| Conversion | Change data types (int, float, base64, etc.) |
| Math | Arithmetic, min/max, random numbers |
| Date and Time | Current time, time zone conversion, date math |
| Referencing | Access trigger/action outputs |
| Workflow | Retrieve flow metadata |
| URI Parsing | Dissect URI strings |
| Manipulation | Work with objects, coalesce, xpath |
